# Capability Tiers

**Day 0 boots. Day 1 works. Day 2 enriches. Every feature must work at Day 1.**

## The Three Days

### Day 0: Boots Without Anything
Binary starts. No config file, no MCPs, no external tools, no network. The TUI renders, the agent loop runs, the REPL accepts input. If it needs something not compiled into the binary, it's not Day 0.

### Day 1: Batteries Included (Standalone)
Built-in tools work. The arch tool analyzes imports. The test tool runs `go test`. The plan tool manages artifact.Graph. The signal bus fires. Watchdogs watch. Gates gate. Trust boundaries inferred from Go imports. Checklists compiled in. All Day 1 features are compiled into the binary. No network, no external processes, no MCP connections required.

This is the factory running on its own equipment.

### Day 2: Ecosystem Connected (MCP Enrichment)
MCP servers connect and upgrade Day 1 built-ins via the same interface:
- Locus replaces built-in arch tool with richer dependency analysis
- Lex replaces built-in checklists with context-resolved rule sets
- Scribe replaces artifact.Graph with persistent cross-session tracking

Each MCP enhances a Day 1 capability. Disconnecting an MCP degrades intelligence but doesn't break functionality.

## Rules

1. **Every feature must work at Day 1.** If it requires an MCP to function, redesign it.
2. **Day 2 enhances, never gates.** MCP down = dumber, not broken.
3. **Same interface, different implementation.** Built-in and MCP tools implement the same port interface. The agent doesn't know which is running.
4. **Day 0 is the smoke test.** If `djinn` doesn't start with zero config, something is wrong.

## Agent Behavior

When designing a feature, ask:
- "Does this work with zero MCPs connected?" → If no, it's not Day 1.
- "What degrades when an MCP disconnects?" → If it breaks, redesign.
- "Can I test this without external infrastructure?" → If no, add a built-in stub.

## Complements

- `factory-principles` — the factory runs on its own equipment (Day 1)
- `pull-not-push` — MCPs connect on demand (Day 2 is JIT)
- `integrate-early` — Day 1 built-ins enable early integration without ecosystem
