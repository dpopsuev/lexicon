# Feature-Driven Development (FDD)

**Core idea:** Organize development around client-valued features. Each feature is small enough to implement in 2 weeks or less.

## Five Processes

1. **Develop an overall model** — high-level walkthrough
2. **Build a features list** — decompose into client-valued functions
3. **Plan by feature** — assign features to developers
4. **Design by feature** — detailed design per feature
5. **Build by feature** — implement, test, integrate

## Feature Format

`<action> the <result> <by|for|of|to> a(n) <object>`

Example: "Calculate the total of a sale"
