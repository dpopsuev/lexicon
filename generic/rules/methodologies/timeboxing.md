# Timeboxing

**Core idea:** Fix the time, flex the scope. Instead of "how long will this take?", ask "what can we deliver in this time?"

## Inversion

| Traditional | Timeboxed |
|---|---|
| Fixed scope, estimate time | Fixed time, prioritize scope |
| Deadline slips | Scope adjusts |
| Everything is P1 | MoSCoW forces prioritization |

## Benefits

- Forces prioritization — most valuable work first
- Prevents scope creep — deadline is non-negotiable
- Reduces Parkinson's Law — work expands to fill time available
- Enables sustainable pace — no death marches

## In Agile

- Scrum sprints (1-4 weeks)
- XP iterations (1-3 weeks)
- DSDM timeboxes with MoSCoW

## Djinn Alignment

- Token budgets are timeboxes — agent has N tokens, flex the scope
