# Kanban

**Core idea:** Visualize work, limit work-in-progress, optimize flow. Pull system — work is pulled when capacity permits, not pushed when requested.

## Core Practices

1. **Visualize the workflow** — board with columns per state
2. **Limit WIP** — cap items per column, prevent overload
3. **Manage flow** — smooth movement, identify bottlenecks
4. **Make policies explicit** — rules for how work progresses
5. **Implement feedback loops** — metrics, retrospectives
6. **Improve collaboratively** — team-driven kaizen

## Key Metrics

- **Lead time** — time from request to delivery
- **Cycle time** — time from work-start to done
- **Throughput** — items completed per time period
- **WIP** — items currently in progress

## Pull vs Push

| Push | Pull |
|---|---|
| Assign work when requested | Pull work when capacity permits |
| Overloads workers | Self-regulating |
| Queue builds up | Queue stays bounded |
