# Scrum

**Core idea:** Fixed-length iterations (sprints, 1-4 weeks) with defined roles and ceremonies.

## Roles

- **Product Owner** — defines what to build (backlog priority)
- **Scrum Master** — removes impediments, facilitates ceremonies
- **Development Team** — self-organizing, cross-functional

## Artifacts

- **Product Backlog** — ordered list of everything needed
- **Sprint Backlog** — items selected for the current sprint
- **Increment** — sum of completed items (potentially shippable)

## Ceremonies

- **Sprint Planning** — what will we do this sprint?
- **Daily Standup** — what did I do, what will I do, any blockers?
- **Sprint Review** — demo the increment to stakeholders
- **Sprint Retrospective** — how to improve the process?

## Key Principles

- Time-boxed sprints (fixed duration)
- No scope changes during sprint
- Definition of Done must be met
- Velocity measured for forecasting
