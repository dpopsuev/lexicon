# Throwaway Prototyping

**Core idea:** Build a quick, cheap prototype to validate requirements and assumptions. Learn from it, then throw it away and build the real system properly.

## When to Use

- Requirements are unclear — prototype to discover them
- UI/UX needs validation before committing to architecture
- Technical feasibility needs proving before full investment

## Anti-Pattern

The throwaway prototype that becomes production. "We'll clean it up later" — you won't. If you can't throw it away, use evolutionary prototyping instead.
