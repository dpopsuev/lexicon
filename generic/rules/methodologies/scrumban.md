# Scrumban

**Core idea:** Combine Scrum's structure (sprints, retrospectives) with Kanban's flow (WIP limits, pull system). Best of both.

## From Scrum

- Sprint cadence for planning and retrospectives
- Roles (optional — can be relaxed)
- Definition of Done

## From Kanban

- WIP limits per column
- Pull system (pull when ready, don't push)
- Continuous flow within the sprint
- Visual board

## When to Use

- Team outgrowing Scrum's rigidity
- Team needing more structure than pure Kanban
- Maintenance teams with unpredictable work
