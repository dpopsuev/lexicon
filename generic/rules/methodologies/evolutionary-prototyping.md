# Evolutionary Prototyping

**Core idea:** The prototype IS the product. Build a robust initial version and continuously refine it based on feedback. Unlike throwaway prototyping, the code is production-quality from the start.

## How It Works

1. Build initial prototype with core functionality
2. Show to users, gather feedback
3. Refine and extend the prototype
4. Repeat until the product is complete

## Key Distinction

| Evolutionary | Throwaway |
|---|---|
| Prototype becomes the product | Prototype is discarded |
| Production-quality code from start | Quick-and-dirty code |
| Iterative refinement | Validate then rebuild |

## Djinn Alignment

Djinn's dogfooding approach IS evolutionary prototyping — the PoC evolves into the MVP evolves into v0.1.0. Same codebase, refined iteratively.
