# Lean Software Development

**Core idea:** Adapted from Toyota Production System. Eliminate waste, amplify learning, deliver as fast as possible.

## 7 Principles

1. **Eliminate waste** — anything not adding customer value
2. **Amplify learning** — iterate, get feedback, run tests
3. **Decide as late as possible** — delay until facts are available
4. **Deliver as fast as possible** — speed enables feedback
5. **Empower the team** — autonomy over micromanagement
6. **Build integrity in** — quality from the start, not bolted on
7. **Optimize the whole** — system thinking, not local optimization

## 7 Wastes of Software Development

| Manufacturing Waste | Software Equivalent |
|---|---|
| Inventory | Partially done work |
| Extra processing | Extra processes (unnecessary bureaucracy) |
| Overproduction | Extra features (gold plating) |
| Transportation | Task switching |
| Waiting | Waiting (for decisions, reviews, builds) |
| Motion | Handoffs between teams |
| Defects | Bugs, rework |
