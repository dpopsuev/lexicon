# Pair Programming

**Core idea:** Two developers work together at one workstation. One writes code (driver), one reviews and thinks ahead (navigator). Roles rotate frequently.

## Benefits

- Continuous code review — defects caught immediately
- Knowledge sharing — no single point of failure
- Better design — two perspectives on every decision
- Fewer interruptions — one person handles while other codes

## Variations

- **Traditional** — same room, same screen
- **Remote** — screen sharing, voice chat
- **Mob programming** — whole team, one screen
- **AI pair programming** — human + AI agent (what Djinn enables)
